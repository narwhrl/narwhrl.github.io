# 为什么要做镜像站？


受不可控因素影响，服务器可能遭受DDOS或遇到宕机等等故障
为了保证大家在主节点故障时能够正常访问网站，利用静态网站托管平台部署了一系列镜像
同时这些平台提供的全球CDN内容分发网络可以提升海外用户的访问体验
然而，以下镜像并不能保证中国大陆内的正常访问，请自备相应的网络工具
每一个镜像都有适合访问的地域，请根据以下的测速情况酌情选择
Cloudflare镜像
# Cloudflare Pages

不论在国内还是国外访问，Cloudflare都可以作为访问的首选镜像
作为全球顶尖的内容分发网络服务商，该镜像可以达到极低的延迟和相应时间
然而由于线路问题，该镜像在国内访问时的响应速度可能不甚理想
但依然可以提供足够稳定的服务和加载速度

Github镜像
# Github Pages

Github作为全球最大的代码托管平台和开源社区
背靠财大气粗的微软，其提供的静态网页托管服务也非常稳定
然而由于众所周知的原因，Github在中国大陆常年受到来自GFW的DNS污染
因此该镜像只推荐从国外或是自备上网工具进行访问

测速详情(点击图片放大)
Github镜像
github.mirror.startrekcn.cn

Vercel镜像
# Vercel

Vercel是一家PaaS公司，著名的Next.JS便是其开发的
其提供的免费静态页面部署以及serverless服务非常良心
虽然该镜像默认分配的域名vercel.app在中国大陆已经无法访问
但载入自己域名之后，该镜像的优秀线路非常适合国内访问
强烈建议国内访问优先使用该镜像

在全球的响应时间表现也不如Vercel和Cloudflare镜像
